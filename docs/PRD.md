# PRD — 입찰 공고 분석기 (Bid Announcement Analyzer)

## 1. 배경 및 목적

선엔지니어링 수주전략팀은 나라장터 입찰 공고문을 매번 수동으로 검토한다.
공고마다 형식이 달라 마감일·참가자격·제출서류를 찾는 데 시간이 소요되고,
복수 공고 비교 및 서류 준비 현황 관리도 어렵다.

**목적:** PDF 공고문을 업로드하면 핵심 정보를 자동 추출·카드 형태로 정리하고,
제출서류 체크리스트와 실적/경력 입력 폼까지 제공해 수주 준비 속도를 높인다.

---

## 2. 대상 사용자

- 선엔지니어링 수주전략팀 담당자 (비개발자 포함)
- 로그인·회원가입 없음, 별도 설치 불필요

---

## 3. 범위 (Scope)

### IN
- PDF 텍스트 추출 + 키워드 매칭 기반 정보 파싱
- 추출 결과 카드 UI
- D-Day 표시 (7일 이내 빨간색)
- 제출서류 체크리스트 + 진행률(%)
- 복수 공고 나란히 비교
- 유사용역 실적·기술자 경력 입력 폼 + 문서 내보내기

### OUT
- 실제 AI/LLM 분석
- 서버·데이터베이스 저장
- 로그인·회원가입
- 나라장터 API 연동

---

## 4. 기능 요구사항

### F-01. PDF 업로드 및 텍스트 추출
- 드래그앤드롭 또는 파일 선택으로 PDF 업로드
- `pdf.js` 등으로 클라이언트 사이드 텍스트 추출
- 추출 실패 시 오류 안내 표시
- **"원문 확인 필요" 경고 배너를 추출 결과 상단에 항상 표시**

### F-02. 핵심 정보 자동 파싱 (키워드 매칭)
추출된 텍스트에서 아래 항목을 정규식·키워드 기반으로 탐색한다.

| 항목 | 탐색 키워드 예시 |
|------|----------------|
| 공고명 | 공고명, 용역명, 사업명 |
| 발주기관 | 발주기관, 수요기관, 공고기관 |
| 예정금액 | 예정금액, 추정가격 |
| 용역기간 | 용역기간, 계약기간, 사업기간 |
| 마감일시 | 마감일, 입찰마감, 제출기한 |
| 참가자격 | 참가자격, 입찰참가자격, 자격요건 |
| 제출서류 | 제출서류, 입찰서류, 구비서류 |

- 탐색 실패 항목은 "미확인 — 원문 확인 필요" 표시

### F-03. 공고 카드 UI
- 파싱 결과를 카드 1장으로 요약
- D-Day 배지: 마감일 기준 잔여 일수, 7일 이내 빨간색, 당일 "D-Day"
- 카드 내 각 항목 옆에 연필 아이콘으로 수동 수정 가능

### F-04. 제출서류 체크리스트
- F-02에서 추출한 제출서류 목록을 체크리스트로 자동 생성
- 각 항목에 완료 체크박스 + "준비 중 / 완료" 상태 토글
- 전체 진행률(%) 프로그레스 바 표시
- 항목 수동 추가·삭제 가능

### F-05. 복수 공고 비교 뷰
- 공고를 2개 이상 업로드하면 가로 나란히 카드 배치
- 최대 4개 공고 동시 표시
- 항목별 가로 정렬로 빠른 비교 가능

### F-06. 실적·경력 입력 폼
- 유사용역 실적: 용역명, 발주기관, 계약금액, 용역기간, 수행내용 (다중 추가)
- 기술자 경력: 성명, 자격증, 주요경력, 투입예정 역할 (다중 추가)
- 폼 작성 후 내보내기 (→ 7. 미결 사항 참고)

### F-07. 데이터 임시 보관
- 작업 내용(공고 파싱 결과, 체크리스트 상태, 폼 입력값)은 `localStorage`에 저장
- 페이지 새로고침 후에도 유지, 브라우저 종료 시 삭제 안 됨
- "데이터 초기화" 버튼 제공

---

## 5. 비기능 요구사항

- 비개발자도 즉시 사용 가능한 단순 UI
- 모든 처리는 클라이언트 사이드 (서버 전송 없음)
- 반응형 레이아웃 (최소 1280px 기준, 모바일은 단일 열)

---

## 5-1. UI 레이아웃 (ERP 대시보드 형태)

### 전체 구조
```
┌──────────┬──────────────────────────────────────┐
│  사이드바  │  메인 패널                            │
│  (220px) │                                      │
│ ⊞ 대시보드│                                      │
│ ≡ 공고목록│                                      │
│ ⇆ 비교   │                                      │
│ ◫ 실적경력│                                      │
│          │                                      │
│ [초기화]  │                                      │
└──────────┴──────────────────────────────────────┘
```

- **사이드바**: 미니멀 고정형 (Notion 스타일), 220px, 페이지 전환 네비게이션 + 데이터 초기화 버튼
- **메인 패널**: 선택된 페이지 컨텐츠 렌더링
- **슬라이드 패널**: 공고 선택 시 우측에서 슬라이드인 (520px), 공고 카드 + 체크리스트 표시, ESC/배경 클릭으로 닫기

### 대시보드 홈
- KPI 카드 3개: 전체 공고 수 / D-7 임박 건수 / 평균 체크리스트 완료율
- 마감 임박 공고 테이블: 공고명, 발주기관, 마감일, D-Day (마감순 정렬, 최대 5건)

### 공고 목록
- 상단: compact 드래그앤드롭 업로드 영역 (항상 표시)
- 테이블 컬럼: 공고명 / 발주기관 / 예정금액 / 마감일 / D-Day 뱃지 / 체크리스트 진행률 / 삭제
- 행 클릭 → 우측 슬라이드 패널 오픈 (선택된 행 하이라이트)
- PDF 업로드 완료 시 공고 목록 탭으로 자동 이동 + 해당 행 자동 선택

## 6. 기술 스택

- **Framework:** Next.js 16 (App Router) + TypeScript
- **Styling:** Tailwind CSS v4
- **PDF 파싱:** `pdfjs-dist` (클라이언트 사이드)
- **내보내기:** `docx` 라이브러리 (Word .docx 다운로드)
- **상태관리:** React useState + localStorage
